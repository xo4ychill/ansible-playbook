# 📊 Ansible Logging Stack: ClickHouse + Vector + Lighthouse

## 🎓 Домашнее задание к занятию 4 «Работа с roles»

Автоматизированное развёртывание стека мониторинга с использованием модульных Ansible‑ролей. Проект полностью параметризован, не содержит хардкода и следует современным DevOps‑практикам управления инфраструктурой.

## 🏗️ Архитектура решения
```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Источники логов│───▶│   Vector Agent  │───▶│  ClickHouse DB  │
│  (stdin/file)   │     │  (сбор/маршрут) │     │ (хранение/анализ)│
└─────────────────┘     └─────────────────┘     └────────┬────────┘
                                                         │
┌─────────────────┐     ┌─────────────────┐     ┌────────▼────────┐
│  Браузер/Клиент │◀─── │   Nginx Proxy   │◀───│  Lighthouse UI  │
│                 │     │ (API/Статика)   │     │ (веб-интерфейс) │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 🔑 Принципы проектирования

- ✅ **Нулевой хардкод**: все IP, пароли, пути и версии вынесены в переменные.
- ✅ **Модульность**: каждый компонент изолирован в отдельной роли.
- ✅ **Идемпотентность**: безопасный повторный запуск без побочных эффектов.
- ✅ **Безопасность**: секреты управляются через ENV / Ansible Vault / External Secrets.
- ✅ **Масштабируемость**: лёгкое добавление новых хостов в группы инвентаря.

## 📂 Структура репозитория
```
ansible-logging-stack/
├── ansible.cfg                  # Глобальная конфигурация Ansible
├── requirements.yml             # Зависимости ролей (внешние + локальные)
├── site.yml                     # Главный playbook с оркестрацией
├── inventory/
│   └── prod.yml                 # Инвентарь (только ссылки на переменные)
├── group_vars/
│   ├── all.yml                  # Глобальные дефолты (ENV/Vault-first)
│   ├── clickhouse.yml           # Переменные группы ClickHouse
│   ├── vector.yml               # Переменные группы Vector
│   └── lighthouse.yml           # Переменные группы Lighthouse
├── host_vars/
│   └── template.yml.example     # Шаблон для хост-специфичных настроек
├── scripts/
│   └── validate-inventory.sh    # Валидация инвентаря перед запуском
├── .env.example                 # Пример файла переменных окружения
├── .gitignore                   # Исключения для Git
└── README.md                    # 📖 Этот файл
```

> 💡 Роли `vector-role` и `lighthouse-role` вынесены в отдельные публичные репозитории согласно требованиям задания и подключаются через `requirements.yml`.

## ⚙️ Требования и подготовка

| Компонент | Минимальная версия | Примечание |
|----------|-----------------|------------|
| ansible | 2.14+ | Требуется поддержка `ansible.builtin.*` модулей |
| python3 | 3.8+ | На контроллере и целевых хостах |
| git | 2.30+ | Для клонирования ролей и репозитория Lighthouse |
| ssh | OpenSSH 8.0+ | Для подключения к хостам |
| curl, jq | Любые актуальные | Утилиты для валидации и healthcheck |

```bash

ansible-galaxy install -r requirements.yml -p roles/

# Проверка корректности установки
